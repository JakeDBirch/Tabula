## Architecture overview

### Layers

Four layers play simultaneously, scheduled by a single shared scheduler:

- **synth** (#a8c5a0) — polyphonic, 16 rows × 16 cols
- **lead** (#b5a0c4) — mono per-column (one note per column max), defaults to +1 octave
- **bass** (#d4a574) — mono per-column, defaults to -1 octave
- **drums** (#c4967a) — 10 voice rows × 16 cols, separate engine

The three synth-type layers (synth/lead/bass) share a single `Bell` audio engine — a custom WebAudio synth class with VCO/VCF/VCA built per-note. Polyphony comes from spinning up a fresh chain per `play()` call. Drums has its own `drumEngine`.

### Per-layer sound design

Lives in `layerParams = {synth, lead, bass}` state. Each slot has its own waveform, detune, ADSR, filter cutoff/res/env, octave, and dlySend. The Bell engine reads these per-call via the `layerP` argument to `play()`. The shared graph stays as one instance — only the per-note params differ.

The active-layer "accessor" pattern: `waveform`, `setWaveform`, `attack`, etc. are local consts that read/write the active layer's slot in `layerParams`. So existing render code that says `<Knob value={waveform} onChange={setWaveform} />` automatically becomes per-layer — no rename needed.

Delay graph itself (TIME, FDBK, HP, LP) is global. Each layer has its own SEND knob into the delay bus. This is the "global delay design with per-layer sends" model.

### Layer store swap mechanism

Only the *active* synth-type layer's pat library lives in the live `pats` state. The other two are kept in `layerStoreR.current[layer] = {pats, activeId, ...}`. When the user switches layers, `switchLayer()` saves the outgoing layer's live state to its slot and loads the incoming layer's slot into live state. The drum layer has its own `drumPats` state and never participates in this swap.

This is a deliberate hack: the original code referenced `pats`/`patsR` in 45+ places, and rewriting them all to `pats[activeLayer]` would have been a massive rewrite. The swap keeps the existing code as-is.

**Consequence**: when the scheduler needs a non-active synth layer's pattern (e.g. it's playing the lead track but synth is the active layer), it has to read from `layerStoreR.current.synth` — NOT from `patsR.current`. The function `playSynthLayerStep(layer, ...)` and the song-mode pat resolution branches handle this. Get this wrong and tracks go silent on layer switch (a bug we hit several times).

### Pattern data model — true polyphony

Every pat has:
- `grid[r][c]` — boolean, is there a note at row r, col c?
- `durs[r][c]` — integer ≥ 1, how many cells does the note at (r,c) extend? Only meaningful where `grid[r][c]` is true. Default 1.
- `params[c]` — per-step params object (vel, flt, dly, rhy, oct, glide). Per-column, not per-cell.
- `gridLen` — how many cols loop (default 16, range 1-16).
- `id`, `name`.

`rhy` is **ratchet only** (1, 2, 3, 4 — number of rapid retriggers in the step). It used to also encode "tie" via rhy=0, but that was column-scoped and broke polyphony. Now durations live on `durs` per-row. Don't ever interpret rhy=0 as a tie — that semantic is dead.

Per-row monophony is enforced in the dur-edit gesture (a row's note can cannibalize same-row notes within its extension). Other rows are independent.

### Song matrix

A 16×16 matrix of pattern references. Rows: 4 row-groups × 4 layers. Cols: 16 bars per group → 64 bars total. Each cell holds a pattern ID for that layer at that bar (or null = silence).

State:
- `songMode` — playback intent. When true, scheduler reads matrix; transport's loop button overrides.
- `songView` — UI gate. When true, matrix is shown; when false, current pattern's note grid is shown. **Decoupled from songMode** so tapping a pill in song view leaves the view without changing playback source.
- `songMatrix = {synth, lead, bass, drums: Array(64)}`
- `songSyncMode` — `true` = unified clock (sync), `false` = per-layer timelines (free).
- `songBar` — current bar in sync mode.
- `songBarLayer = {synth, lead, bass, drums}` — per-layer cursor positions. In sync mode, mirrored from songBar. In free mode, updated independently.

### Sync vs Free song modes

**Sync** (default): one `s` clock advances all layers. Bar duration = min `gridLen` of populated pats this bar. Single playhead column lights up across all four layers.

**Free**: each layer has its own `{step, nextAt, bar}` in `freeR.current`. They start aligned at play but drift apart based on each pattern's gridLen. Each layer loops within its own populated bar range. Empty layer = silent. Synth has the all-empty fallback (loops its active pat at bar 0 when the entire matrix is empty).

The free-mode scheduler is a separate early-exit branch at the top of `scheduler()`. It calls `playSynthLayerStep()` and `playDrumStep()` helpers — same audio code as sync, but driven per-layer instead of by a unified clock.

### UI rules

- **Layer bar**: tap inactive = switch. Tap active = open sound drawer.
- **Pattern pills**: outside song view, tap inactive = activate, tap active = open drawer (jumps to STEP page). In song view, any tap = exit view + show that pattern's grid.
- **SONG chip**: 3 visual states. Lit when in view. Dim-lit when songMode is on but view is off (e.g. after pill tap). Unlit when off.
- **FOLLOW** auto-disables on grid edits and step-lane edits (so user editing isn't disrupted by playhead-following).
- **Bottom chips**: TEMPO · SONG · VARY · PROJECT. SOUND chip and SEQUENCE chip are gone — they're reachable via tap-active-layer / tap-active-pattern respectively.

## Critical lessons (don't relearn these)

### Mobile grid pointer events

The note grid uses `onPointerDown` on the parent gridRef container, NOT on each cell. After mobile redesign in early 2025 the grid stopped working because cells had become individual event targets — fix was reverting to plain `<div>` cells with the parent container handling everything. **If you change grid event handling, test on iPhone immediately.**

### `return_react2` artifact viewer bug

The old artifact viewer used a CommonJS Babel transform that emitted `return_react2` (an undefined identifier) when it encountered **module-level arrow functions returning JSX**. Symptoms: blank artifact, ReferenceError in console. Fix: inline all JSX directly — never extract to a top-level `const X = () => <jsx>`.

This bit primarily on the artifact viewer, not the deployed site, but the build pipeline includes a CJS audit pass that greps for `return_react2` to catch it early. Don't remove the audit.

### useCallback empty deps trap

`handleGridDown = useCallback(..., [])` baked in the FIRST render's closures over `pushHistory` etc. Subsequent edits used stale references. Fix: ref-based callbacks. Define `pushHistoryR = useRef(()=>{})`, reassign `pushHistoryR.current = ...` every render, and have stable callbacks invoke `pushHistoryR.current()`. Same pattern for `captureSnapshotR`. Don't simplify these back to direct closures.

### iOS gesture interception

Parent containers with `touch-action: pan-x` or `overflow-x: auto` cause iOS Safari to intercept all gestures starting in that region at the OS level. Vertical drags don't propagate. If a row's touches feel "stuck horizontally," check the parent's `touch-action`.

### Babel only catches syntax errors, not undefined refs

When extracting variables (e.g. `const H = ...` → `cell`/`gridW`/`gridH`), Babel happily compiles undefined references. They only fail at runtime. **Always grep for the old name everywhere before building.**

### Drum default symbol

The very first drum pat must use `mkDrumPat(symPat(0))`, not `mkDrumPat("A")`. The "A" was leftover from before the celestial-glyph naming pool. Other layers use symPat(0) so drums should match.

### Stale projects with dlyWetPct / waveform / etc

Old saves predate per-layer params. The load path migrates flat fields (waveform, detune, ADSR, dlyWetPct, etc.) into the synth slot of `layerParams`; lead and bass get defaults. Same migration is in `applyShareState`. Don't remove these fallbacks.

### Don't push without explicit instruction

The user says "push it" when they want a deploy. Until then, builds stage to `index.html` for testing in the artifact viewer / locally but DO NOT git commit + push. Several times early in development I built changes, claimed they were deployed, but had only staged them locally — confusing for the user when the live site didn't reflect them. **Never claim "pushed" or "deployed" without having actually run `git push`.**

### Save/load coverage

When you add new state, audit FOUR sites and add it to all of them:
1. `doSave` (snap object)
2. `doLoad` (apply branch)
3. `getShareState` / `applyShareState`
4. `captureSnapshotR.current` / `applySnapshot` (undo/redo)

Forgetting #4 means edits don't undo cleanly. Forgetting #1-3 means saves silently lose data.

### Cross-layer audio bugs

The synth main path used to look up patterns in `patsR.current` — that's only correct when synth is the active layer. The lead/bass branches already used `layerStoreR.current[layer]` correctly, but synth was the odd one out. Same shape of bug existed in the old non-song-mode chain lookup. Whenever you read pat data in the scheduler, confirm you're using the right source for the layer.

## Build pipeline

```bash
npm install         # one-time: installs @babel/core, @babel/cli, @babel/preset-react, @babel/preset-env
npm run build       # compiles src/tabula.jsx → index.html
npm run audit       # runs the CJS return_react2 audit standalone
```

The build does:
1. Strip `import React, ...` and convert `export default function Tabula` → `function Tabula`
2. Append a `ReactDOM.createRoot(...).render(...)` line
3. Run Babel with preset-env + preset-react
4. Wrap in HTML scaffold with React 18 UMD CDN scripts and PWA meta tags
5. Run a CJS-mode Babel pass to grep for `return_react2` (returns nonzero if found)

GitHub Pages serves `index.html` from `main` automatically.

## Things that exist as ballast

A few state fields and helpers are still in the code as compatibility ballast for old saves. They don't drive any UI. Leave them alone unless you're doing a dedicated cleanup pass:

- `synthPhrases`, `drumPhrases`, `sections`, `activeSynthPhraseId`, `activeDrumPhraseId`, `activeSectionId` — leftover from the pre-song-matrix arrangement system.
- `chain`, `setChain` — the legacy synth-track chain. Scheduler no longer reads it in non-song mode (each layer plays its own active pat instead).
- `recMode`, `setRecMode` — variation self-record state. No UI to toggle it currently.
- `collapseToMono` — function defined but unused.
